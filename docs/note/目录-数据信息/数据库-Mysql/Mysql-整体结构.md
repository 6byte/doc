### Mysql-架构

#### 连接池

```
原理
    A.系统初始化时，将数据库连接作为对象存储在内存中
    B.当用户访问数据库时，不建立新连接，而是从连接池取出空闲连接对象
	C.使用完后将连接放回连接池中，以供下一个请求访问使用
	D.连接的建立、断开都由连接池自身来管理。
	E.设置参数能控制 初始连接数、连接的上下限数、最大使用次数、最大空闲时间

最小连接数：
	连接池一直保持的数据库连接
	可能造成资源被浪费

最大连接数：数据库连接请求超过此数，新请求将等待
```
